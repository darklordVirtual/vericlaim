runtime. In Fig. 2 (middle), we vary the block size of FlashAttention, which results in different amounts of HBM accesses, and measure the
runtime of the forward pass. As block size increases, the number of HBM accesses decreases (as we make fewer
passes over the input), and runtime decreases. For large enough block size (beyond 256), the runtime is then bottlenecked by
