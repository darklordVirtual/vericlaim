SWE-bench task instance example. Problem statement is an aggregation of the issues related to the pull request. Codebase corresponds to a repository and base commit. The tests and solution are derived from the original PR’s associated code changes. Stylized for readability. Execution-based validation. We verify the usability of a task instance via execution. For each candidate, we first define a virtual environment to serve as an execution context, then install before applying any patches, and finally run once before and once after the solution is applied. A candidate is removed from consideration for the final dataset if any step in the verification process fails. In addition, to ensure that a solution is non-trivial, we compare the pre-solution and post-solution validation logs to check for whether there are one or more tests in where the status changes from fail to pass.
