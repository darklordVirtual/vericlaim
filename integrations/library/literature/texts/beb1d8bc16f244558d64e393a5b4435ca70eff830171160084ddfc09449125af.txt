For every top-level comment (comments that have no parent comment), we create a sequence of comments by traversing down the comment tree from the top-level comment. We choose the next comment by taking the child comment with the highest number of children comments (a cheap attempt at taking a long path through the comment tree, note that it does not take the longest possible path). We consider all stories that have at least one comment and are not flagged by the moderators for potential conduct violations. Since comments are stored in HTML, we use the html2text package to extract the text from the post. We order each document by listing the title, url, sub-title, and author at the top. Top-level comments are delimited by "\n------\n" and sub-comment chains are delimited by "\n~~~\n". We include author and extracted text for each comment.
