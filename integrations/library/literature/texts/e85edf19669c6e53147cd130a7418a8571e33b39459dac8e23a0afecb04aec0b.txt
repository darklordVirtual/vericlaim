Alongside the task instances, we also create a corresponding folder containing the ground truth test results. For each task instance, from their respective and test-to-status mappings, we create a test results data structure where the keys are FAIL_TO_FAIL, FAIL_TO_PASS, PASS_TO_FAIL, and PASS_TO_PASS, and the values are lists of tests. By “caching” these results, we remove the need to re-run the solution at evaluation time (although re-running is an available option). We use this data structure to verify task completion, as discussed in Section A.4.
