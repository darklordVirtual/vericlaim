While each of id, entity, time, and attributes is OPTIONAL, at least one of them MUST be present. A reference to a given entity MAY appear in multiple usages that share
a given activity identifier. Example 21 ◊

The following usages

used(a1, e1, 2011-11-16T16:00:00, [ ex:parameter="p1" ])
used(a1, e2, 2011-11-16T16:00:01, [ ex:parameter="p2" ])

state that the activity identified by a1 used two entities identified by e1 and e2, at times 2011-11-16T16:00:00 and 2011-11-16T16:00:01, respectively; the first
one was found as the value of parameter p1, whereas the second was found as value of parameter p2. The semantics of parameter is application specific. 5.1.5 Communication

Communication ◊ is the exchange of some unspecified entity by two activities, one activity using some entity generated by the other. A communication implies that activity a2 is dependent on another a1, by way of some unspecified entity that is generated by a1 and used by a2.
