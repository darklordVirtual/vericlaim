Learning unknown model classes or hyper-parameters. For our online attacks, we obtained information about the model class of , the enabled feature extraction ex, and other hyper-parameters, directly from the ML service or its documentation. More generally, if does not have full certainty about certain model characteristics, it may be able to narrow down a guess to a small range. Model hyper-parameters for instance (such as the free parameter of an RBF kernel) are typically chosen through cross-validation over a default range of values. Given a set of attack strategies with varying assumptions, can use a generic extract-and-test approach: each attack is applied in turn, and evaluated by computing or over a chosen set of points. The adversary succeeds if any of the strategies achieves a low error. Note that needs to interact with the model only once, to obtain responses for a chosen set of extraction samples and test samples, that can be re-used for each strategy.
