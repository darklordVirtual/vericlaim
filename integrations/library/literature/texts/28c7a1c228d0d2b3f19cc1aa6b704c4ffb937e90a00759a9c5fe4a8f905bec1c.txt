Actions the rule floor does not decide are escalated to a semantic judge: a strong language model prompted to read the action in context and return a verdict together with a rationale and a confidence. The judge is the component that reads intent, and it is the component that handles the semantic threats rules cannot — on the mainly-semantic-attack corpus of §5 it nearly doubles rule accuracy with near-zero false-blocks, and it does so across model families and two providers (§5.2). Recall is not the scarce resource here: even the smallest judge we test catches the semantic attacks the rules miss (§5.5). What scales with model capability is precision — the rate at which the judge over-escalates a benign action. Because the judge can over-escalate, AgentTrust v2 never acts on a raw judge verdict. The judge’s output is admitted only through a confidence gate: an escalation (an allow warn/block/review move) is taken only when the judge’s confidence clears a threshold ; below the action keeps the rule floor’s verdict.
