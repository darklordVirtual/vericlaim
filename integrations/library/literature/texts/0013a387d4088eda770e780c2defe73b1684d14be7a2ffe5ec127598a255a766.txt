The semantic class cannot be served by simply caching the judge’s verdicts. The natural idea — remember the verdict for an action and reuse it when the action recurs — fails by construction on exactly this class, because the lookup key is a surface that semantic twins share: a verdict-cache keyed on the action retrieves the wrong precedent for the benign-vs-malicious pair it is meant to separate, and has no high-precision operating regime at any coverage (the negative result of §5.4, Figure 4). The cache fails for the same reason the rule floor fails — it keys on surface. This memory is the semantic generalization of v1’s judge cache: v1 shipped an exact content-hash plus incremental-delta cache that amortized token cost on identical, recurring contexts [Yang 2026, §4.8]; the semantic store retrieves by meaning for accuracy, not by hash for cost — the same component, evolved from cheaper-on-repeat to smarter-on-semantic. AgentTrust v2 instead uses retrieval as evidence for the judge, not as a replacement for it: a guarded RAG memory.
