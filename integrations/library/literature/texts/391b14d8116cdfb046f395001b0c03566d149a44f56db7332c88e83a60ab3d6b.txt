For example, an activity that describes the relocation of an entity will have start and end locations, as well as every place in between those points. Example 49 ◊

5.7.2.3 prov:role

A role ◊ is the function of an entity or agent with respect to an activity, in the context of a usage, generation, invalidation, association, start, and end. The attribute prov:role is allowed to occur multiple times in a list of attribute-value pairs. The value associated with a prov:role attribute MUST be a PROV-DM Value. Example 50 ◊

The following activity is associated with an agent acting as the operator. wasAssociatedWith(a, ag, [ prov:role="operator" ])
