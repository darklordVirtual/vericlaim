To minimize the risk of any data contamination, the set of repositories in the training data are disjoint from the packages included in the evaluation benchmark. Training details. Given the instructions, an issue text from GitHub and the relevant code files as the prompt, we finetune SWE-Llama to generate the patch that solved the given issue (the “gold patch”). For memory efficiency, we fine-tune only the weights of the attention sublayer using LoRA Hu et al. (2022),
and exclude training sequences with more than tokens, reducing the effective size of the training corpus to instances. More details are provided in Appendix B.
